## Objective

Turn the successful single FastAPI application into a small modular application framework.

Prove it with a real multi-module Research Notes application.

## Complete user journey

The user must be able to run:

```bash
servicefabric apps create research-notes \
  --template modular-web-app

servicefabric apps modules research-notes
servicefabric apps build research-notes
servicefabric apps start research-notes
servicefabric apps status research-notes
servicefabric apps resources research-notes
servicefabric apps open research-notes
```

Then:

```bash
servicefabric call notes.create \
  --input '{"title":"Gateway design","body":"Applications expose selected actions."}'

servicefabric call notes.search \
  --input '{"query":"gateway"}'
```

The browser interface must also create and search notes through the same application service logic.

Stopping the API module alone must make its capabilities unavailable. Restarting it must not require rebuilding the whole application.

## Demonstration application

Add:

```text
examples/research-notes/
```

It must contain:

```text
one normal React or Svelte interface
one FastAPI service
one persistent local storage module
one capability adapter
```

Choose React or Svelte according to the best-maintained existing template. Document the choice.

Do not depend on the component lab. The frontend must be a standard framework application.

Publish:

```text
notes.search
notes.get
notes.create
```

Display their business sensitivity:

```text
notes.search = read only
notes.get = read only
notes.create = creates application data
```

## Module standard

Define the smallest useful shared module description covering:

```text
module identity and version
module kind
provided interfaces
required interfaces
configuration references
storage references
health
resource expectations
application membership
capabilities provided
startup and shutdown relationship
```

Modules must communicate through documented interfaces, not through imports of one another’s internal implementation.

## Framework kits

Add only kits proven by the application:

```text
FastAPI service kit
chosen frontend framework kit
persistent local storage kit
Python capability-adapter kit
```

Each kit should provide shared support for:

```text
configuration
application and module identity
health reporting
structured logs
safe errors
capability registration
operation correlation
```

Keep generated projects ordinary framework projects that standard coding assistants can understand.
