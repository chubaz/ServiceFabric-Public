## Programme objective

ServiceFabric has two connected roles:

1. It builds, imports, hosts and operates applications and reusable modules.
2. It publishes selected actions from hosted or external applications as controlled capabilities for CLIs, workflows and AI consumers.

This milestone must prove both roles with one small real application.

Do not build isolated framework infrastructure that has no usable user journey.

## Required user journey

A new user must be able to run:

```bash
servicefabric apps install examples/text-utility
servicefabric apps build text-utility
servicefabric apps start text-utility
servicefabric apps status text-utility
servicefabric apps resources text-utility
servicefabric tools list
servicefabric tools describe text.count_words
servicefabric call text.count_words \
  --input '{"text":"ServiceFabric hosts applications and capabilities."}'
servicefabric apps stop text-utility
```

After the application is stopped, calling `text.count_words` must return a clear unavailable response and must not execute the application action.

The commands must be real subprocess commands using an installed `servicefabric` executable. They must not require the user to construct Python objects.
