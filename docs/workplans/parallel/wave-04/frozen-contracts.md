# Wave-4 Frozen Contracts

The following remain unchanged during specialist work: `ToolDefinition`, AP-01A capability behavior, canonical package/application/module contracts, workspace persistence boundaries, `EffectContract` in `servicefabric_contracts.effects`, and the separation between canonical runtime contracts and consumer projections.

Any change requires an explicit contract-change decision, integration approval, a new bootstrap base, and lane synchronization. Wave-4 introduces definitions and static registration only; it does not introduce invocation or runtime availability.
