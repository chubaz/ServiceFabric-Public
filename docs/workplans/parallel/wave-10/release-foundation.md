# Foundation-release surface

Release readiness presents current canonical ServiceFabric behavior as the product. It provides `servicefabric doctor`, concise installation and a five-minute demonstration, a product architecture overview, a foundation capability reference, an honest maturity and limitations statement, and a machine-readable foundation-release manifest.

Legacy numbered services may be named only as contained compatibility context; they are not documented as the primary architecture. Doctor checks local prerequisites and declared package integrity without provisioning resources, installing dependencies, or exposing secrets.
