# Wave-5 Integration Order

1. Freeze the Wave-5 boundaries and launch specialist lanes from the bootstrap commit.
2. Review availability and its module-health source protocol.
3. Review the loopback HTTP adapter independently of invocation composition.
4. Review invocation against the accepted availability and transport protocols.
5. Compose the application-status bridge and CLI in the integration lane.
6. Integrate the single acceptance journey after the composed CLI is available.
7. Run `make verify-wave-05` once after candidate composition.
8. Run `make verify-current` once at final closure, record completion, and prepare the canonical handoff.

Candidate commits are reviewed explicitly and are never merged automatically.
