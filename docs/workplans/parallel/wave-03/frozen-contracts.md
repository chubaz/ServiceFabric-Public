# Wave-3 Frozen Contracts

The following existing public boundaries are frozen for specialist work:

- application/module manifests and primitive compatibility;
- application assembly plans and dependency ordering;
- resource-binding resolution and persistence;
- reviewed framework-kit and blueprint references;
- AP-00C managed process control;
- workspace layout and artifact-store APIs;
- the thin CLI dispatch boundary.

The minimum new shared boundaries are deterministic generator output, immutable build-manifest fields, and guidance-fragment composition. A change after specialist launch requires a Contract Change Request, integration approval, a committed decision, a new integration base, and synchronization of affected lanes.
