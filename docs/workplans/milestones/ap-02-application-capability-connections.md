# AP-02: Application Capability Connections

## Objective
Generalise reviewed application capability connections after hosting and resource boundaries are proven.

## Starting conditions
AP-01A, AP-00, and AP-01B are merged with passing regressions.

## Scope
Author, validate, discover, and operate selected application actions through canonical invocation, governance, approval, operation, and projection boundaries.

## Exclusions
No automatic endpoint scraping, implicit tool publication, gateway-owned business logic, public multi-tenant service, or arbitrary remote inventory trust.

## Acceptance and rollback
Multiple consumers use one reviewed application capability without importing its implementation. Revert commits; canonical records remain independently removable local fixtures.
